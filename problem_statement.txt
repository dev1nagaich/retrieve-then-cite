The goal
Build a small “retrieve-then-cite” prototype. It reads a couple of textbook PDFs, answers a few questions
using only what is inside those books, and tells us exactly which passage each answer came from. Then it
measures how often it found the right source.
What we give you
• 2 textbook PDFs (attached). Physiotherapy if you have them; any 2 books are fine.
• 5 test questions, each with the page or section where the correct answer lives.
What to build
A simple pipeline — a command-line script is perfectly fine:
1 Read the 2 PDFs into text. Handle the case where a page is a scanned image, not real text.
2 Split the text into smaller pieces (“chunks”), and attach to each chunk where it came from (book name,
chapter or page).
3 Embed each chunk — turn it into a vector (a list of numbers that captures meaning) — using any
embedding model you like.
4 Store and search: keep the vectors somewhere searchable and, for a question, fetch the top few most
relevant chunks.
5 Answer with a citation: use any LLM to write a short answer using only the fetched chunks, and have it
state which chunk or page it used.
6 Measure: run the 5 test questions and report a simple number — for how many did the correct source
appear in your top results.
What to send back
• A GitHub repo or zip with runnable code and a short README (how to run it).
• A one-page note: the choices you made and why (chunk size, how many results you fetch, which
embedding model and LLM), what you would improve with more time, and anything that broke.
• The measurement output: the number, plus 2 example answers showing their citations.
Tools and ground rules
• Use whatever you know — FAISS, Chroma, any vector store; any embedding model; any LLM (OpenAI,
open-source, anything). You do not need our exact stack. We care about how you think, not brand
names.
• Keep API keys and secrets out of the repo.
•
If something is unclear, make a reasonable assumption and write it in the README.
• Respect your own time — this is a one-day slice, not a finished product.

What we are not judging
Fancy UI, perfect accuracy, or whether you used our cloud. We are judging your method, your reasoning,
and whether you think about reliability.
